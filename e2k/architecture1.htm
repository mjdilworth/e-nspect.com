e-nspect2000 Architecture Samples

The following sections illustrate two possible architectures.
Illustrates a simple architecture, where there is one of everything.
Data is collected from the SMTP server and inserted into the Database Server. The e-nspect client is used to query the database server and to configure the e-nspect components on the Exchange server.

Illustrates a more complex architecture where there are multiple Exchange SMTP Servers and multiple database servers.

Data is collected from multiple Exchange servers, some of which is entered into a common database on one database server and some into another database on another database server. The e-nspect clients can manage all this from a single point because of the flexibility of the MMC framework.
