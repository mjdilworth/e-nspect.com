e-nspect2000 High Usage Installation

The following details the necessary steps for undertaking an e-nspect2000 installation for a High Usage usage production system.
For the e-nspect2000 High Usage Installation we are making the following assumptions:
  1. You have either one or multiple Microsoft Exchange 2000 Server machines.
  2. You have an existing SQL7 or SQL2000 server installation. (Alternatively, you could use the SQL Desktop Engine)
  3. You have a management workstation.
Under such circumstances you have to:
  1. If needed run the e-nspectDB setup program to install the SQL2000 Desktop Engine. On the designated SQL server machine
  2. Run the e-nspect2k setup program on every Exchange 2000 Server that you wish to monitor.
    • Select to install the Exchange Components.
    • Provide, when prompted, the name of the SQL server machine.
    • Provide, when prompted, the SQL Server username (by default the e-nspectDB setup establishes the user account "sa" , no quotes).
    • Provide, when prompted, the SQL Server user's password (by default the e-nspectDB setup establishes a blank password for the "sa" account , no quotes).
    • Provide, when prompted, the location of a temporary file system folder.
  3. Run the e-nspect2k setup program on every Management Workstation.
    • Select to install the Client Tools. On one workstation select also the Database Services.
  4. On the workstation. Run the client tools menu option (Start->Programs->e-nspect2000->Client Tools).
  5. Enjoy :)
