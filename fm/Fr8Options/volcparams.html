Volatility Calculator - Input & Output Parameters

Below we briefly describe the input and output parameters of the Volatility Calculator:-

Date (Column A),Data Column B, Cell B3: The program allows the user to cut and paste (or manually type) into the appropriate columns the dates and data of an index or route that is being calculated/studied. Cell B3 allows the user to manually input a title to the column for sake of clarity. In our example the user is calculating the volatility of the BHMI spot market.

Column C: This column gives the daily, weekly, monthly (depending on users input in column A) volatility of price 'returns' in percentage terms and NOT the volatility of prices.

Calculation Parameters: Here the user (via the drop down menu) is given the choice of calculating volatility by the 'moving average' method or by the 'exponentially weighted moving average method'.

Moving Average: Here the user need only input a figure for the 'Time Window' the 'EWMA decay factor' is N/A.

Exponentially Weighted Moving Average (EWMA): The user is not required to add any input as EWMA uses the 'EWMA decay factor'* function only.

* Note: FreightMetrics uses a 'decay factor' of 0.94 as research has shown that this number is suitably accurate for freight volatility.

Correlation Calculator - Input & Output Parameters

The input parameters of the Correlation Calculator are similar to the parameters of the Volatility Calculator as shown above. The obvious difference is in Column B and Column C where the user adds the data (through 'cut and paste' or manually) that will be calculated/studied for correlation. As discussed above the correlation will be of price 'returns' in percentage terms and NOT the correlation of prices in absolute terms. Cells B3 and C3 are customisable to allow a user to input their own meaningful title for sake of clarity.
